C++ std::nullopt

一、基本概念

简单来说，std::nullopt 是一个哨兵值（Sentinel Value），专门用来表示 std::optional 处于“无值”（空）的状态。

在底层，std::nullopt 的定义非常简单：

  • 它的类型是 **std::nullopt_t**（一个空的强类型结构体）。
  • std::nullopt 是 std::nullopt_t 的一个 inline constexpr 常量。

std::optional 重载了接受 std::nullopt_t 的构造函数和赋值运算符。因此，当你把 std::nullopt 赋给 std::optional 时，它知道应该将内部状态标记为“空”。

二、使用场景

  1. 作为函数返回值（表示失败或无结果）

  2. 显示初始化

  3. 重置一个 std::optinonal 对象

  4. 条件对比

三、std::nullopt vs nullptr vs NULL

特性 std::nullopt nullptr NULL
所属标准 C++17 C++11 C++98 / C 语言遗留
设计目的 表示 std::optional 为空 表示空指针（不指向任何内存） 表示空指针（宏定义，通常是 0）
对应类型 std::nullopt_t std::nullptr_t 整型 0 或 (void*)0
安全性 极高（强类型，防止各种隐式转换） 高（解决 NULL 的整型重载二义性） 低（容易引发重载混乱和安全隐患）

为什么不能用 nullptr 代替 std::nullopt？

因为 std::optional 保存的是值，而不是指针。 如果一个 std::optional 保存的是一个指针，那么：

  • opt = nullptr; 意味着：opt 有值，但这个值是一个空指针（*opt 的结果是 nullptr）。
  • opt = std::nullopt; 意味着：opt 没有值（你连指针都没有，尝试 *opt 会触发未定义行为或抛出异常）。

这就是类型安全的美妙之处，std::nullopt 完美规避了这种语义模糊。

  1. 写返回值时：如果函数可能无法产生值，返回 std::nullopt 比返回魔术值（如 -1、空字符串 ""）要安全和优雅得多。
  2. 清空对象时：opt = std::nullopt; 比 opt = {}; 语义更清晰。
  3. 不要滥用比较：虽然 if (opt != std::nullopt) 可行，但通常直接写 if (opt) 或 if (opt.has_value()) 更符合 C++ 的现代习惯。
